1.주석이란 무엇이며, 종류는? 
: 프로그래밍에 대한 보충 설명이 필요할때 기입한다.
- 블록 주석 : /*(시작) 내용 */(마감)
- 행 주석 : // 내용
 
2.주석은 컴파일시 어떻게 되는가?
: 대상에서 제외된다.

3.들여쓰기는 왜 해야 되는가?
: 변수가 어떤 항목의 범위에 종속되는지를 쉽게 표현할 수 있다.

4.변수란 무엇인가?
: 메모리 공간을 활용하기 위한 도구이다.

5.8형제(자료형)을 써보세요(feat.외우세요).
: 
boolean / 1byte / 참,거짓 / true, false
char / 2byte / 문자 / 유니코드
byte / 1byte / 정수 / -128 ~ 127
short / 2byte / 정수 
int / 4byte / 정수 / -21억 ~ 21억
long / 8byte / 정수 / 
float / 4byte / 실수 /
double / 8byte / 실수 /

6.변수선언의 의미는 무엇인가?
: 메모리 공간의 할당을 의미하며 변수의 용도와 이름을 결정짓는다.

7.int 형 범위는 어떻게 되는가?
: 약 -21억 ~ +21억 (2의 32제곱이지만 맨 앞자리는 부호가 들어가므로 실제로는 2의 31제곱)

8.int num; 을 메모리로 설명해 보세요.
: 메모리에 4바이트 크기의 int가 생성되고 2의 31제곱까지의 정수가 입력된다.

9.문자가 뿌려지는 원리에 대하여 설명하시오.
: 각 문자마다 약속된 숫자를 지정하고 인코딩&디코딩 과정을 거쳐 출력한다.  

10.아스키 코드란 무엇인가요?
: 영문이나 특수기호를 표현하기 위해 컴퓨터가 인식할 수 있는 기계어로 표준화한 것이다.
